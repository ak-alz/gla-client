# Growth Layer Agent

Лёгкий кросс-платформенный десктоп-агент, который считает, сколько времени
вы провели сосредоточенно, а сколько — отвлекаясь, и отдаёт эту картину
**вам самим** — как личный дневник продуктивности, не как отчёт для
начальника у вас за спиной.

Этот репозиторий — не рекламный текст о приватности. Это исходный код,
который её реализует. Ниже — конкретные факты о том, что агент делает и
не делает, с точными ссылками на код, который это доказывает. Не верьте
на слово — прочитайте сами, для этого репозиторий и публичный.

## Почему не «слежка»

Слежка — это когда собирают всё, что могут технически достать, и решают,
что с этим делать, потом. Здесь наоборот: список того, что вообще
разрешено собирать, зафиксирован в коде заранее, и всё, что в этот
список не входит, физически не читается — не «скрывается после сбора»,
а не считывается вообще.

- **Никогда не читается текст заголовка окна** — только категория,
  выведенная из имени процесса (`chrome.exe`/`chrome` → «браузер»,
  `code.exe`/`code` → «IDE» и т.д., на Windows и Linux соответственно).
  Классификация по имени процесса —
  [`crates/normalization/src/categories.rs`](crates/normalization/src/categories.rs).
  Есть отдельный, специально изолированный модуль
  [`title_classifier.rs`](crates/normalization/src/title_classifier.rs),
  который умеет прочитать заголовок вкладки браузера и превратить его в
  категорию (чтобы отличить, например, работу в Jira от чтения новостей) —
  но в реально собранном агенте он подключается с пустым набором правил
  и поэтому не активируется никогда; сам заголовок этот код не
  возвращает, не логирует и не сохраняет ни при каких обстоятельствах —
  это зафиксировано прямо в его докстринге.
- **На GNOME (Wayland) имя процесса узнаётся через маленькое,
  собственное Shell-расширение** —
  [`installer/linux/gnome-extension/`](installer/linux/gnome-extension/),
  клиент — [`gnome_extension.rs`](crates/linux-collector/src/gnome_extension.rs).
  Существует только потому, что `org.gnome.Shell.Eval` закрыт в unsafe
  mode с GNOME 41 и другого способа узнать активное окно не осталось.
  Расширение отдаёт по D-Bus ровно то же самое, что X11/Hyprland-код
  отдаёт на своих платформах — id приложения и PID, ничего больше;
  весь его исходный код — меньше 60 строк, читается целиком за минуту.
- **Никогда не считывается содержимое ввода** — ни какая именно клавиша
  нажата, ни координаты курсора/клика. Только счётчики «нажатий»/
  «движений мыши»/«кликов» за интервал. Реализация системных хуков —
  [`crates/windows-collector/src/hooks.rs`](crates/windows-collector/src/hooks.rs).
- **Скриншоты не делаются вообще** — в кодовой базе нет ни одной строчки
  кода, которая умела бы захватить экран.
- **Пауза — реальная, не фиктивная.** Кнопка «Приостановить» в трее
  останавливает сам опрос активности, не только отправку на сервер — во
  время паузы `collector.poll()` не вызывается ни разу
  ([`main.rs`](crates/agent-bin/src/main.rs), `run_collector_loop`).
  Состояние паузы видно даже без открытия меню — цвет иконки в трее
  меняется (`crates/ui-shell/src/tray.rs`).
- **Ничего лишнего в трей не встроено.** Агент никогда не создаёт ни
  одного окна — ни для настроек, ни для дашборда, ни для чего-либо
  ещё. Единственный интерфейс — иконка в трее с коротким меню
  (статус/пауза/диагностика/ссылка на дашборд/выход). Полноценный
  дашборд с графиками открывается в браузере, не встраивается сюда.

## Что реально уходит на сервер

Ровно то, что видно в [`event-contract`](crates/event-contract/) — секунды
активности/простоя по категориям, счётчики ввода, отрезки непрерывной
работы одной категории и их длительность, число незакрытых «провалов»
активности. Опционально (управляется флагом `app_detail`) — то же самое, но
с точностью до имени процесса, а не только категории (напр. отдельно
«code.exe» и «chrome.exe» внутри категории «IDE»/«браузер»). Ни заголовков
окон, ни содержимого нажатий, ни скриншотов в этом списке нет структурно —
не потому, что их решили не отправлять, а потому, что их негде взять: их
просто не читает ни один коллектор.

## Честно про то, что ещё не готово

Публичный код — значит и предупреждения из него никуда не спрятать.
На сегодня:

- Флаги согласия (`Consent { .. }` в `main.rs`) **захардкожены** при
  сборке, не настраиваются пользователем в этой версии — переключаемый
  UI для них ещё не реализован.
- Токен привязки устройства хранится в `config.json` как обычная
  JSON-строка, не в системном хранилище секретов (Windows Credential
  Manager/Secret Service) — код для этого в репозитории есть
  (`crates/secrets/`), но агент пока его не использует.
- Удаление агента не удаляет локальные данные (очередь, лог, `config.json`
  с токеном) — это сознательное решение (чтобы переустановка не рвала
  привязку), но значит, что для полного удаления нужно вручную стереть
  папку данных (см. `crates/agent-bin/src/paths.rs`).
- Автозапуск включается автоматически при установке, не по отдельному
  согласию.
- macOS — черновик кода, ни разу не собирался и не запускался на
  реальном железе (см. `crates/macos-collector/`).
- KDE (Wayland) — активное окно пока не определяется вообще (нужен
  отдельный KWin-скрипт по образцу GNOME-расширения выше), сознательно
  отложено, не забыто. X11, Hyprland и GNOME уже работают.
- GNOME-расширение выше проверено на одной реальной установке
  (Ubuntu, GNOME 50) — работает, но не тестировалось на диапазоне
  версий GNOME 45–52, который оно заявляет как совместимый.

## Технически

Rust workspace: платформенные коллекторы (`windows-collector`/
`linux-collector`/`macos-collector`) за общим интерфейсом
(`collector-core`) — `linux-collector` сам определяет окружение (X11,
Hyprland по IPC, GNOME через собственное Shell-расширение) и честно
отдаёт статус «недоступно», а не гадает, там где ничего из этого не
подошло — общая нормализация сигналов (`normalization`),
надёжная локальная очередь на диске на случай офлайна (`durable-queue`),
устойчивая выгрузка с бэкоффом (`uploader`), подписанные обновления с
атомарным откатом (`updater`/`update-manifest`), сервисный жизненный цикл
— единственный экземпляр, восстановление после сбоя, автозапуск,
реакция на сон/блокировку экрана (`lifecycle`) — и минимальный трей-шелл
(`ui-shell`), который никогда не создаёт ни одного окна.

## Лицензия

Код помечен как `UNLICENSED` — это осознанно: репозиторий публичный для
того, чтобы его можно было прочитать и проверить, а не для того, чтобы
его можно было форкнуть и выдать за отдельный продукт. Если нужны права
на переиспользование — обсуждается отдельно.
