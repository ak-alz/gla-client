# Установка

Проще всего — скачать готовый установщик со страницы
[Releases](../../releases). Ниже — что именно он делает на каждой
платформе, и как собрать агент из исходников самому (например, чтобы
свериться, что скачанный файл собран из того же кода).

## Текущие версии

Номер версии здесь намеренно не указан текстом — он устаревает при
каждом релизе (уже наступали на эти грабли в паре других мест этого
проекта). Актуальный — всегда на странице [Releases](../../releases),
единственном источнике правды; `crates/agent-bin/Cargo.toml`'s
`version` — то же самое число в исходниках.

| Платформа | Статус | Формат | Проверено вживую |
|---|---|---|---|
| Windows 10/11 (x64) | ✅ Готов | Inno Setup `.exe`, без прав администратора | Установка/обновление/удаление, трей без консоли, меню, реальная выгрузка данных |
| Linux (x64), Ubuntu/Debian | ✅ Готов | `.deb` | Установка, автозапуск, трей, GNOME-расширение, реальная выгрузка данных — всё проверено на реальной машине (Ubuntu, GNOME 50) |
| Linux (x64), Fedora/RHEL | ⚠️ Собран, не проверен | `.rpm` | Пакет собирается (`rpmbuild`), но не устанавливался на настоящий RPM-дистрибутив |
| Linux (x64), Arch | ✅ Готов | `PKGBUILD` (AUR-style, следует `git` HEAD) | Собирается и устанавливается в реальном Arch-контейнере |
| Linux (x64), любой дистрибутив | ✅ Готов | `.tar.gz` + `install.sh`, без пакетного менеджера | Установка/автозапуск проверены |
| macOS | 🚧 Черновик кода | `.app`/`.dmg` (скрипты есть) | Не собирался, не подписывался, не нотаризован — нет macOS-железа и Apple Developer-аккаунта |

## Windows

Скачать `.exe` с [Releases](../../releases) и запустить — установка не
требует прав администратора. Данные кладутся в
`%LOCALAPPDATA%\GrowthLayerAgent\`, ярлык — только в меню Пуск (без
иконки на рабочем столе). Автозапуск регистрируется агентом при первом
старте. После установки: трей-иконка → «Привязать устройство» —
откроется дашборд с уже готовым кодом.

Собрать самостоятельно:

```powershell
cargo build -p agent-bin --release
& "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer\windows\agent.iss
# -> dist\windows\GrowthLayerAgentSetup-<version>.exe
```

## Linux

Готовые команды «скачать и поставить одной строкой» — на
[странице загрузки](https://devpace.ru/download) сайта. Ниже — как
собрать из исходников самому:

```bash
cargo build -p agent-bin --release

# .deb (Debian/Ubuntu)
bash installer/linux/deb/build.sh
sudo apt install -y ./installer/linux/deb/dist/growth-layer-agent_*.deb

# .rpm (Fedora/RHEL — собирается, но не проверен на реальном хосте)
bash installer/linux/rpm/build.sh
sudo dnf install -y ./installer/linux/rpm/dist/growth-layer-agent-*.rpm

# Arch (следует HEAD этого репозитория, не фиксированный тег)
cd installer/linux/arch && makepkg -si

# Любой дистрибутив без пакетного менеджера — .tar.gz + install.sh
bash installer/linux/tarball/build.sh
tar xzf installer/linux/tarball/dist/growth-layer-agent-*.tar.gz
cd growth-layer-agent-*/ && ./install.sh
```

Все четыре способа устанавливают в `/usr/bin/growth-layer-agent` (или
`~/.local/bin/` для tarball), сами добавляют пользователя в группу
`input` (нужно для счётчиков клавиатуры/мыши — `evdev_counter.rs`),
регистрируют автозапуск (`systemd --user` юнит + XDG autostart) и сразу
запускают агент. Данные — в `$XDG_DATA_HOME/growth-layer-agent` (обычно
`~/.local/share/growth-layer-agent`). Есть также скрипт подписанного
APT-репозитория (`installer/linux/repo/build_apt_repo.sh`) для тех, кому
нужен полноценный `apt install` вместо разового `.deb`.

**Активное окно/процесс** (нужно для категорий вместо «Другое»): X11 —
через EWMH, работает всегда; Hyprland — через его собственный IPC-сокет,
работает всегда; **GNOME (Wayland)** — через собственное
[Shell-расширение](installer/linux/gnome-extension/), которое все четыре
установщика ставят и включают сами — после установки нужен один выход
из системы и повторный вход, GNOME подхватывает новые расширения только
при перезапуске Shell; KDE (Wayland) — пока не реализовано (нужен
отдельный KWin-скрипт, в планах, см. README).

**Иконка в трее** — отдельный от расширения выше вопрос: показывает её
`gnome-shell-extension-appindicator` (пакеты подключают её сами, где
доступно) на стоковом GNOME, KDE/XFCE/Cinnamon её уже показывают
нативно.

## macOS

Код есть (`crates/macos-collector`, `installer/macos/*.sh`,
`com.growthlayer.agent.plist`), но никогда не собирался и не
запускался на реальном железе — честно отмечен как черновик, не
скрытая готовность.

## Как проверить, куда агент отправляет данные

Собранный из `cargo build --release` (то же самое, что производят все
установщики) по умолчанию идёт на боевой адрес продукта; только debug-
сборка (`cargo build` без `--release`) по умолчанию указывает на
`http://localhost:8000` (см. `config.rs`'s `#[cfg(debug_assertions)]`
split). В обоих случаях адрес можно переключить, отредактировав
`backend_url` в `config.json` в папке данных агента (пути выше) и
перезапустив агент — без переустановки. Сам факт того, что этот адрес
лежит в обычном текстовом файле, а не зашит в бинарник, — тоже часть
проверяемости: можно убедиться, куда именно уходят данные, не разбирая
исполняемый файл.

## Полное удаление

Штатное удаление (через «Установка и удаление программ» на Windows,
`dpkg -r`/`apt remove` на Linux) останавливает агент и убирает
автозапуск, но **не трогает папку данных** (это сознательное решение,
чтобы переустановка не рвала привязку устройства — см. README). Чтобы
стереть всё до конца, дополнительно удалите:

- Windows: `%LOCALAPPDATA%\GrowthLayerAgent\`
- Linux: `~/.local/share/growth-layer-agent/` (или `$XDG_DATA_HOME/growth-layer-agent`, если переопределён)
