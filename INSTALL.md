# Установка

Проще всего — скачать готовый установщик со страницы
[Releases](../../releases). Ниже — что именно он делает на каждой
платформе, и как собрать агент из исходников самому (например, чтобы
свериться, что скачанный файл собран из того же кода).

## Текущие версии

| Платформа | Версия | Статус | Формат | Проверено вживую |
|---|---|---|---|---|
| Windows 10/11 (x64) | 0.1.2 | ✅ Готов | Inno Setup `.exe`, без прав администратора | Установка/обновление/удаление, трей без консоли, меню, реальная выгрузка данных |
| Linux (x64) | 0.1.2 | ✅ Готов | `.deb` | Ubuntu: установка, автозапуск, трей без окна, реальная выгрузка данных |
| Linux (x64) | 0.1.2 | ⚠️ Собран, не проверен | `.rpm` | Пакет собирается (`rpmbuild`), но не устанавливался на настоящий RPM-дистрибутив (Fedora/RHEL) |
| macOS | — | 🚧 Черновик кода | `.app`/`.dmg` (скрипты есть) | Не собирался, не подписывался, не нотаризован — нет macOS-железа и Apple Developer-аккаунта |

Версия = `crates/agent-bin/Cargo.toml`'s `version` (единый источник
правды — `.deb`/`.rpm` берут её оттуда же скриптом сборки; `agent.iss`'s
`MyAppVersion` и `main.rs`'s `AGENT_VERSION` держатся синхронно вручную
при каждом бампе). Чек-суммы каждого релиза — на странице
[Releases](../../releases), рядом с файлами.

## Windows

Скачать `.exe` с [Releases](../../releases) и запустить — установка не
требует прав администратора. Данные кладутся в
`%LOCALAPPDATA%\GrowthLayerAgent\`, ярлык — только в меню Пуск (без
иконки на рабочем столе). Автозапуск регистрируется агентом при первом
старте. После установки: трей-иконка → «Привязать устройство» —
откроется дашборд с уже готовым кодом.

Собрать самостоятельно:

```powershell
cargo build -p agent-bin --release
& "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer\windows\agent.iss
# -> dist\windows\GrowthLayerAgentSetup-<version>.exe
```

## Linux

```bash
# .deb (Debian/Ubuntu)
cargo build -p agent-bin --release
bash installer/linux/deb/build.sh
sudo dpkg -i installer/linux/deb/dist/growth-layer-agent_*.deb

# .rpm (Fedora/RHEL — собирается, но не проверен на реальном хосте)
bash installer/linux/rpm/build.sh
```

Устанавливается в `/usr/bin/growth-layer-agent` (без демонов/systemd-
юнитов от пакета — автозапуск агент регистрирует сам, как per-user
`systemd --user` юнит, при первом запуске из-под обычного пользователя).
Данные — в `$XDG_DATA_HOME/growth-layer-agent` (обычно
`~/.local/share/growth-layer-agent`). Есть также скрипт подписанного
APT-репозитория (`installer/linux/repo/build_apt_repo.sh`) для тех, кому
нужен полноценный `apt install` вместо ручного `dpkg -i`.

Требует X11 или Wayland-сессию с работающим system-tray host
(StatusNotifierWatcher) — GNOME/KDE Wayland такого не предоставляют «из
коробки» и явно не поддерживаются (см. `linux-collector`'s
`environment.rs`); Hyprland и обычный X11 — поддерживаются.

## macOS

Код есть (`crates/macos-collector`, `installer/macos/*.sh`,
`com.growthlayer.agent.plist`), но никогда не собирался и не
запускался на реальном железе — честно отмечен как черновик, не
скрытая готовность.

## Как проверить, куда агент отправляет данные

По умолчанию — на `http://localhost:8000` (локальный dev-backend).
Чтобы переключить на прод-адрес, отредактировать `backend_url` в
`config.json` в папке данных агента (пути выше) и перезапустить —
без переустановки. Сам факт того, что этот адрес лежит в обычном
текстовом файле, а не зашит в бинарник, — тоже часть проверяемости:
можно убедиться, куда именно уходят данные, не разбирая исполняемый файл.

## Полное удаление

Штатное удаление (через «Установка и удаление программ» на Windows,
`dpkg -r`/`apt remove` на Linux) останавливает агент и убирает
автозапуск, но **не трогает папку данных** (это сознательное решение,
чтобы переустановка не рвала привязку устройства — см. README). Чтобы
стереть всё до конца, дополнительно удалите:

- Windows: `%LOCALAPPDATA%\GrowthLayerAgent\`
- Linux: `~/.local/share/growth-layer-agent/` (или `$XDG_DATA_HOME/growth-layer-agent`, если переопределён)
